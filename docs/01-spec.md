# UNIV SITDOWN 기능명세서

> 버전: 1.0  |  작성일: 2026.04.22  |  대상: 백엔드 개발

## 1. 문서 개요

### 1.1 목적
대학교 내 열람실, 강의실, 스터디룸, PC실 등의 좌석을 효율적으로 관리하고, 실시간 좌석 예약 및 이용 현황을 제공하는 모바일 애플리케이션 **UNIV SITDOWN**의 기능 요구사항을 정의한다.

### 1.2 범위
본 명세서는 백엔드 서버가 제공해야 하는 기능, 주요 비즈니스 규칙, 상태 정의, 데이터 모델, 비기능 요구사항을 포함한다. 화면 UI는 플러터 담당자가 별도 작업하며, 본 문서에서는 화면 ID와 그에 대응하는 기능 범위만 명시한다.

### 1.3 목표
- 대학교 내 좌석 자원의 가시성 확보 및 이용 효율 향상
- 실시간 혼잡도 기반 예약, 연장, 취소 기능 제공
- 공간과 좌석을 행/열 단위로 동적으로 구성·운영할 수 있는 구조 제공
- 개인별 이용 통계 제공을 통한 공간 이용 습관 개선 유도

### 1.4 용어 정의

| 용어 | 정의 |
|---|---|
| 공간(Space) | 열람실, 강의실, 스터디룸, PC실 등 좌석을 제공하는 단위 공간 |
| 좌석(Seat) | 공간 내에서 행(Row)·열(Column) 좌표로 구분되는 개별 이용 단위 |
| 예약(Reservation) | 특정 좌석을 일정 시간 동안 점유하기 위한 요청/계약 기록 |
| 혼잡도 | 공간의 현재 또는 시간대별 이용률 (여유/보통/혼잡) |
| 연장 | 기존 예약의 종료 시간을 추가 시간만큼 연장하는 행위 |
| 관리자 | 공간·좌석의 생성/수정/삭제 및 공지 등록이 가능한 권한 사용자 |

### 1.5 사용자 역할

| 역할 | 구분 | 권한 범위 |
|---|---|---|
| 학생(User) | 일반 사용자 | 좌석 조회, 예약/연장/취소, 이용 통계 조회, 공지 열람 |
| 관리자(Admin) | 운영자 | 공간·좌석 생성/수정/삭제, 공지 등록, 전체 예약 현황 관리 |
| 게스트(Guest) | 비로그인 | 로그인/회원가입 화면 접근만 가능 |

## 2. 화면 구성

본 애플리케이션은 총 8개의 주요 화면 그룹으로 구성되며, 인증 → 홈 → 검색 → 예약 → 마이 흐름을 따른다.

| 화면 ID | 화면명 | 카테고리 | 주요 내용 |
|---|---|---|---|
| SCR-01 | 홈 | 메인 | 내 예약 요약, 빠른 예약, 실시간 혼잡도, 추천 좌석 |
| SCR-02 | 공간 검색 | 검색 | 공간 목록, 카테고리 필터, 혼잡도 |
| SCR-03 | 강의실 상세 | 상세 | 공간 정보, 운영시간, 좌석 수, 특징, 혼잡도 예측 |
| SCR-04 | 좌석 예약 (배치 확인) | 예약 | 행·열 좌석 배치도, 상태별 색상 |
| SCR-05 | 좌석 선택 (상세 정보) | 예약 | 선택 좌석 정보, 이용 시작/종료 시간 |
| SCR-06 | 예약 확인 | 예약 | 예약 완료 안내 |
| SCR-07 | 예약 내역 | 내 정보 | 진행 중 / 지난 예약 / 취소 내역 탭 |
| SCR-08 | 마이 (내비게이션) | 내 정보 | 내 정보 / 내 예약 / 이용 통계 / 즐겨찾기 / 공지 / 설정 |
| SCR-08-1 | 내 정보 | 내 정보 | 이름, 아이디, 이메일, 전화번호, 소속 |
| SCR-08-2 | 내 예약 | 내 정보 | 현재 예약, 연장/취소, 예약 예정 |
| SCR-08-3 | 내 이용 통계 | 내 정보 | 주간/월간/연간 이용 시간, 주요 이용 공간 |
| SCR-08-4 | 공지사항 | 내 정보 | 전체/안내/점검/이벤트 탭 |
| SCR-08-5 | 설정 | 내 정보 | 알림 설정, 다크모드, 언어 |
| SCR-08-6 | 앱 정보 | 내 정보 | 서비스 소개, 이용약관, 개인정보 처리방침 |
| SCR-08-7 | 로그아웃 | 내 정보 | 로그아웃 확인 |
| SCR-09~12 | 로그인 플로우 | 인증 | 로고 → 이메일 → 비밀번호 → 완료 |
| SCR-13~16 | 회원가입 플로우 | 인증 | 이메일 입력 → 인증 → 정보 입력 → 완료 |

## 3. 기능 명세

### 3.1 인증 / 회원 (AUTH)

| 기능 ID | 기능명 | 관련 화면 | 기능 설명 |
|---|---|---|---|
| F-AUTH-01 | 이메일 로그인 | SCR-10, SCR-11 | 이메일과 비밀번호를 입력하여 로그인한다. 형식 검증 및 오류 메시지를 제공한다. |
| F-AUTH-02 | 비밀번호 찾기 | SCR-11 | 등록된 이메일로 비밀번호 재설정 링크를 발송한다. |
| F-AUTH-03 | 이메일 인증 회원가입 | SCR-13, SCR-14 | 이메일 입력 후 6자리 인증 코드를 발송·확인한다. 제한 시간(3분) 카운트다운. |
| F-AUTH-04 | 회원 정보 입력 | SCR-15 | 이름, 비밀번호, 비밀번호 확인을 입력하며 정책(길이/특수문자)을 검증한다. |
| F-AUTH-05 | 회원가입 완료 | SCR-16 | 가입 완료 안내 후 홈으로 진입한다. |
| F-AUTH-06 | 자동 로그인 | SCR-09 | 유효 토큰이 존재하면 로그인 단계를 건너뛰고 홈으로 이동한다. |
| F-AUTH-07 | 로그아웃 | SCR-08-7 | 세션/토큰을 제거하고 로그인 화면으로 이동한다. |

### 3.2 홈 (HOME)

| 기능 ID | 기능명 | 기능 설명 |
|---|---|---|
| F-HOME-01 | 사용자 환영 메시지 | 로그인된 사용자 이름을 포함한 환영 문구 표시 |
| F-HOME-02 | 내 예약 요약 카드 | 진행 중 예약(공간, 좌석, 시간, 남은 시간)을 상단 노출 |
| F-HOME-03 | 빠른 예약 / 최근 예약 | 최근 예약 이력 기반 1회 탭 재예약 경로 제공 |
| F-HOME-04 | 실시간 혼잡도 그래프 | 대표 공간의 시간대별 혼잡도를 라인 차트로 표시 |
| F-HOME-05 | 추천 좌석 카드 | 태그(조용함/창가/콘센트 등) 기반 추천 좌석 1건 노출 |

### 3.3 공간 검색 / 상세 (SPACE)

| 기능 ID | 기능명 | 기능 설명 |
|---|---|---|
| F-SPACE-01 | 공간 통합 검색 | 키워드로 공간·강의실 검색 |
| F-SPACE-02 | 카테고리 필터 | 전체 / 열람실 / 스터디룸 / PC실 탭으로 필터링 |
| F-SPACE-03 | 공간 목록 카드 | 썸네일, 공간명, 혼잡도, 운영시간, 좌석 수, 특징 표시 |
| F-SPACE-04 | 공간 상세 조회 | 운영 시간, 좌석 수, 특징(콘센트/조용/와이파이 등) 제공 |
| F-SPACE-05 | 혼잡도 예측 그래프 | 시간대별 혼잡도 예측을 막대 그래프로 제공 |
| F-SPACE-06 | 좌석 선택하기 진입 | 상세 화면 하단 버튼으로 좌석 배치도로 이동 |
| F-SPACE-07 | 공간 즐겨찾기 | 별 아이콘 토글로 즐겨찾기 추가/해제 |

### 3.4 좌석 예약 (RSV)

| 기능 ID | 기능명 | 기능 설명 |
|---|---|---|
| F-RSV-01 | 좌석 배치도 렌더링 | 행(1~8) × 열(1~10)을 동적으로 생성하며, 공간 규격에 따라 크기 조정 |
| F-RSV-02 | 좌석 상태 색상 표시 | 사용 가능(초록) / 사용 중(회색) / 선택 불가(연회색) / 선택(파랑) 구분 |

| F-RSV-05 | 이용 시간 설정 | 시작/종료 시간을 시간 선택기로 설정, 이용 시간 자동 계산 |
| F-RSV-06 | 예약 유효성 검증 | 운영 시간 이탈, 중복 예약, 최대 이용 시간 초과 검증 |
| F-RSV-07 | 예약 생성 | 서버에 예약을 생성하고 완료 화면으로 전환 |
| F-RSV-08 | 예약 완료 안내 | 예약 정보 요약 표시, 홈 이동 / 예약 내역 보기 제공 |

## 4. 상태 정의

### 4.1 좌석 상태

| 상태 | 표시 색상 | 의미 / 행동 |
|---|---|---|
| `AVAILABLE` | 초록 | 사용 가능. 선택하여 예약 가능 |
| `OCCUPIED` | 진한 회색 | 다른 사용자가 현재 이용 중. 선택 불가 |
| `UNAVAILABLE` | 연한 회색 | 고장/점검/관리자 설정으로 사용 불가. 선택 불가 |
| `SELECTED` | 파랑 | 현재 사용자가 선택한 좌석. 예약 확정 전 |
| `RESERVED` | 연한 파랑 | 해당 시간대에 다른 사용자 예약 존재. 선택 불가 |

### 4.2 예약 생명주기

| 상태 | 전이 조건 | 설명 |
|---|---|---|
| `PENDING` | 예약 생성 직후 | 예약 완료 화면 노출 전의 임시 상태 |
| `SCHEDULED` | 예약 확정, 시작 전 | 예약 예정 목록에 노출 |
| `IN_USE` | 시작 시간 도달 | 홈 카드 및 내 예약 진행 중 탭에 노출 |
| `EXTENDED` | IN_USE 중 연장 성공 | 종료 시간 갱신 후 IN_USE로 간주 |
| `COMPLETED` | 종료 시간 도달 | 지난 예약 내역으로 이동 |
| `CANCELED` | 사용자/관리자 취소 | 취소 내역 탭에 노출 |
| `NO_SHOW` | 시작 후 일정 시간 미체크인 | 자동 취소 및 패널티 부여(옵션) |

## 5. 비즈니스 규칙

| 규칙 ID | 항목 | 내용 |
|---|---|---|
| BR-01 | 동시 예약 제한 | 사용자당 진행 중 예약 1건, 예정 예약 최대 1건까지 허용 |
| BR-02 | 최대 이용 시간 | 단일 예약은 최대 4시간까지 설정 가능 (공간 정책으로 재정의 가능) |
| BR-03 | 연장 허용 | 종료 10분 전부터 최대 2시간 범위 내 연장, 충돌 시 불가 |
| BR-04 | 취소 정책 | 시작 전 취소 제약 없음. 이용 중 취소 즉시 반영 |
| BR-05 | 운영 시간 | 운영 시간(예: 06:00~22:00) 외 예약 생성 불가 |
| BR-06 | 중복 예약 방지 | 동일 시간대 동일 좌석에 복수 예약 생성되지 않도록 서버에서 락 처리 |
| BR-07 | 노쇼 처리 | 시작 후 15분 내 체크인 미수행 시 자동 취소 및 경고 1회 (정책 가변) |
| BR-08 | 혼잡도 산정 | 점유 좌석 수 / 전체 사용 가능 좌석 수 기준 — LOW <40%, NORMAL 40~75%, HIGH >75% |

## 6. 비기능 요구사항

| 구분 | 항목 | 요구사항 |
|---|---|---|

## 7. 데이터 모델 (논리)

아래 모델은 Java + Spring Boot + JPA 기반 구현 기준의 논리 모델이며, 실제 물리 스키마는 PostgreSQL 벤더와 인덱스 전략에 따라 조정될 수 있다.

### 7.1 User

| 필드 | 타입 | 설명 |
|---|---|---|
| id | UUID | 사용자 고유 식별자 |
| email | String (unique) | 로그인 이메일 |
| passwordHash | String | 해시된 비밀번호 (BCrypt) |
| name | String | 이름 |
| phone | String | 전화번호 |
| affiliation | String | 소속 (예: 학생) |
| profileImageUrl | String? | 프로필 이미지 URL |
| role | Enum | `USER` / `ADMIN` |
| createdAt | LocalDateTime | 가입 일시 |

### 7.2 Space

| 필드 | 타입 | 설명 |
|---|---|---|
| id | UUID | 공간 고유 식별자 |
| name | String | 공간명 (예: 제1열람실) |
| floor | int | 층수 |
| category | Enum | `READING_ROOM` / `STUDY_ROOM` / `PC_ROOM` / `LECTURE_ROOM` |
| openTime | LocalTime | 운영 시작 시간 |
| closeTime | LocalTime | 운영 종료 시간 |
| maxReservationHours | int | 단일 예약 최대 시간 |
| features | List\<String\> | 콘센트/조용/와이파이 등 태그 |
| thumbnailUrl | String? | 목록용 이미지 |

### 7.3 Seat

| 필드 | 타입 | 설명 |
|---|---|---|
| id | UUID | 좌석 고유 식별자 |
| spaceId | UUID | 소속 공간 ID (FK) |
| row | int | 행 번호 (1부터) |
| column | int | 열 번호 (1부터) |
| label | String | 표시용 라벨 (예: A-12) |
| isEnabled | boolean | 사용 가능 여부 (관리자 설정) |
| features | List\<String\> | 좌석 단위 태그 |

### 7.4 Reservation

| 필드 | 타입 | 설명 |
|---|---|---|
| id | UUID | 예약 고유 식별자 |
| userId | UUID | 예약자 ID (FK) |
| seatId | UUID | 대상 좌석 ID (FK) |
| startAt | LocalDateTime | 시작 일시 |
| endAt | LocalDateTime | 종료 일시 |
| status | Enum | `SCHEDULED` / `IN_USE` / `COMPLETED` / `CANCELED` / `NO_SHOW` |
| canceledAt | LocalDateTime? | 취소 일시 |
| extendedCount | int | 연장 횟수 |
